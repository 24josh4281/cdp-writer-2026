# Evaluation Engine Batch Improvements - Plan

## Executive Summary

| 관점 | 요약 |
| --- | --- |
| 문제 | 현재 평가는 D/A/M/L 충족 가능성을 보여주지만, 어떤 기준 때문에 득점 또는 감점됐는지 문항별로 추적하기 어렵습니다. |
| 해결 | 평가기준을 조건 단위로 풀어 표시하고, 응답/증빙 문장과 연결하며, D/A/M/L 예상 점수 요약을 함께 보여줍니다. |
| 사용자 효과 | 컨설턴트가 사전평가표를 검토할 때 감점 사유와 보완 문장을 빠르게 확인할 수 있습니다. |
| 핵심 가치 | 향후 10개 이상 기업 평가에서도 판단 근거의 일관성과 검토 속도를 높입니다. |

## Batch Scope

| 기능 | 목적 | 실행 방식 |
| --- | --- | --- |
| 조건별 평가 상세 | 평가기준별 충족/미충족 사유를 표시 | 각 D/A/M/L 카드 안에 조건, 상태, 근거 문장을 표로 표시 |
| 예상 점수 요약 | 문항별 D/A/M/L 예상 점수 확인 | 기준별 충족률과 배점 정보를 사용해 예상 분자/분모 표시 |
| 평가 Export 보강 | 검토 산출물 재사용성 향상 | CSV/JSON에 조건별 상태, 예상 점수, 신뢰도를 포함 |
| 공식 채점 구조 감지 | route, best row, row-level, partial scoring 문구를 표시 | 점수 계산 참고 메모로 노출 |
| 필수조건 점검 | A/A- 등급 상한 리스크 확인 | 전환계획, Scope, 검증, 목표, 거버넌스 조건 점검 |
| 산업특수 문항 | 선택 섹터에 맞는 특수 문항 후보 표시 | sector tag와 원문 키워드 기반 필터링 |
| XLSX Export | 기존 사전평가 엑셀 흐름과 연결 | 전체 문항, 감점 문항, 필수조건, 산업특수, 배점요약, 증빙매핑 시트 생성 |

## Risks And Assumptions

- 자동 점수는 공식 CDP 점수 확정이 아니라 사전평가 보조입니다.
- 복잡한 route, best row scoring, sector-specific 예외는 별도 고도화가 필요합니다.
- 증빙 문장 매핑은 키워드 기반 1차 근거이며, 최종 판단은 평가자가 원문과 대조해야 합니다.
- GPT 생성 답변도 규칙 기반 D/A/M/L 점검을 다시 통과해야 합니다.

## Success Criteria

- 평가 카드에서 조건별 `충족 가능 / 부분 보완 필요 / 보완 필요` 상태가 보입니다.
- 각 조건마다 확인된 근거 문장 또는 부족 사유가 표시됩니다.
- 문항별 D/A/M/L 예상 점수와 신뢰도가 보입니다.
- CSV/JSON Export에 평가 상세 정보가 포함됩니다.
- XLSX Export에 6개 사전평가 시트가 생성됩니다.
- 기존 로컬 실행과 공개 정적 링크가 깨지지 않습니다.
