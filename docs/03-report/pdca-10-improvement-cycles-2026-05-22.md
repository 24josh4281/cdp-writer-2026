# CDP Writer 2026 - PDCA 10회 개선 사이클 결과

작성일: 2026-05-22

## 목적

10개 이상의 기업 응답을 반복 평가할 수 있도록, 평가 정확도와 운영 안정성을 높이는 소규모 개선을 10개 사이클로 나누어 적용했다.

## 10개 개선 사이클

| Cycle | Plan | Do | Check | Act |
|---|---|---|---|---|
| 1 | API 연결 상태를 사용자가 바로 이해해야 함 | `/api/health` 결과를 화면 상태 문구와 연결 | health 응답 확인 | GPT 가능/키 없음/토큰 필요 여부를 표시 |
| 2 | GPT 생성 실패 원인을 명확히 해야 함 | GPT 생성 전 health preflight 추가 | OPENAI_API_KEY 미설정 상태에서 안내 오류 확인 | 파일 추출/XLSX는 계속 사용 가능하다고 분리 안내 |
| 3 | API 토큰이 브라우저 저장소에 남는 보안 리스크 제거 | workspace 저장 항목에서 `apiToken` 제외 | 저장 스냅샷에서 토큰 제외 확인 | 토큰은 입력 중인 세션에서만 사용 |
| 4 | 전체 평가 전 품질 수준을 한눈에 봐야 함 | 평가 화면에 품질 게이트 패널 추가 | 작성 커버리지/감점/필수조건/신뢰도/증빙 길이 계산 확인 | 평가 전 보완 우선순위를 제시 |
| 5 | 증빙 품질 부족으로 인한 오판을 줄여야 함 | 증빙 원문 길이, 파일 추출 성공/실패, 초안 수 점검 추가 | 빈 증빙 상태에서 부족/검토 표시 확인 | 증빙 추가 액션을 화면에 표시 |
| 6 | 감점 문항이 많을 때 어디부터 볼지 정렬 필요 | 검토 우선순위 큐 추가 | severity와 score gap 기반 정렬 확인 | 상위 감점 후보를 먼저 검토 |
| 7 | 최대 득점 경로를 문항별로 내보내야 함 | `최대득점경로` XLSX 시트 추가 | atomized condition 기반 행 생성 확인 | 작성자가 조건별로 응답을 보완 가능 |
| 8 | 기존 사전평가 산출물과 연결되는 시트 확장 필요 | `검토우선순위`, `품질게이트`, `API상태` 시트 추가 | XLSX 생성 API 검증 | 업무 산출물에 QA 로그를 포함 |
| 9 | CSV/XLSX 수식 주입 리스크 제거 필요 | `=`, `+`, `-`, `@` 시작 셀에 apostrophe prefix 적용 | 테스트 XLSX에서 formula tag 없음 확인 | 외부 파일 공유 시 보안 리스크 감소 |
| 10 | 서버 상태/제한값이 불명확함 | health 응답에 API version과 업로드 제한 추가 | `2026.05.22-pdca10` health 확인 | 운영 중 문제 발생 시 원인 파악 쉬움 |

## 남은 리스크

- GPT 생성은 `OPENAI_API_KEY`가 설정되어야 활성화된다.
- 자동 평가는 여전히 사전평가 보조 도구이며, 최종 제출 전 CDP 원문 평가방법론과 사람이 대조해야 한다.
- 산업 특수 문항 적용은 sector tag와 키워드 기반 후보 필터이므로, 기업의 실제 CDP 산업분류와 최종 대조가 필요하다.
