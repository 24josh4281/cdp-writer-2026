# CDP 평가 정확도 개선 1~8 실행 보고

작성일: 2026-05-22

## P — Plan

목표는 앞으로 10개 이상 기업 응답을 반복 평가할 수 있도록, 자동평가 결과의 설명력과 검토 가능성을 높이는 것이다. 특히 평가방법론과 실제 사전평가 엑셀 간 점수 차이를 줄이기 위해 채점 조건, 증빙 위치, 필수조건, 산업특수 문항, 사람 검토 의견을 한 흐름에서 관리하도록 했다.

## D — Do

1. 공식 채점 규칙 DB화
   - 문항별 D/A/M/L 평가기준을 원자화 조건과 route, best row, row-level, partial 여부로 정리했다.
   - XLSX export에 `채점규칙DB` 시트를 추가했다.

2. 증빙 위치 추적 강화
   - DOCX, PPTX, XLSX, PDF 추출 결과에 문서/슬라이드/시트 행/페이지 위치 태그를 붙였다.
   - 문항별 증빙 매핑 시트에서 어떤 조건이 어떤 증빙으로 충족됐는지 확인할 수 있게 했다.

3. 방법론 동기화 승인 기능 고도화
   - CDP Portal export 파일 분석 결과를 문항별 변경 후보로 보여준다.
   - 후보마다 `승인`, `보류`, `제외`를 선택하고 승인 항목만 overlay로 적용하도록 변경했다.

4. 기업별 프로젝트 관리
   - 프로젝트 메뉴를 추가해 회사별 초안, 증빙, 벤치마크, 리뷰 메모를 분리 저장/불러오기 할 수 있게 했다.

5. 정확도 검증용 벤치마크 세트
   - 기존 사전평가 XLSX/CSV를 업로드해 문항번호와 D/A/M/L 점수 표현을 자동평가 결과와 비교한다.
   - XLSX export에 `벤치마크비교` 시트를 추가했다.

6. 필수조건 전용 평가 화면
   - A/A- 등급 필수조건을 별도 메뉴에서 한 번에 점검하도록 분리했다.

7. GPT 작성 품질 통제
   - GPT 또는 규칙 기반 초안에서 증빙에 없는 수치, 목표, 검증 표현이 감지되면 `[증빙 필요]`와 `[검토 필요]`를 표시한다.
   - 서버 API에도 같은 품질통제 지시를 전달한다.

8. 리뷰어 모드
   - 자동 감점 후보별로 사람이 `보류`, `인정`, `수정 필요`, `제외` 판단과 메모를 남길 수 있게 했다.
   - XLSX export에 `리뷰의견` 시트를 추가했다.

- Route A/B, best row, sector-specific logic은 계속 실제 사례로 보정해야 한다.
